01 C · Structures de données · Analyse

Comparaison de
Structures de Données

Étude comparative en C de plusieurs structures de données : tableaux statiques, tableaux dynamiques et listes chaînées. Implémentation complète de chaque structure, mesure expérimentale des performances, et analyse des compromis selon les cas d'usage.

Ce que j'ai fait

Implémentation rigoureuse des trois structures. Protocole de benchmark reproductible. Comparaison des coûts d'insertion, suppression et accès selon la taille des données.

Limites honnêtes

Tests limités à des jeux de données synthétiques. Pas de benchmarks multi-threadés.

Ce que j'ajouterais

Intégration d'arbres et de tables de hachage pour un panorama plus complet. Visualisation des résultats via matplotlib.

03 Web · Flask · PostgreSQL · Base de données

Application Web
SPA — Refuges

Conception et développement complet d'une application web pour la gestion des refuges de la SPA. Du schéma entité-association jusqu'au site fonctionnel : modélisation, implémentation PostgreSQL, et interface Flask permettant aux employés de gérer les animaux, les soins et les transferts entre refuges.

Ce que j'ai fait

Modélisation relationnelle complète sous PostgreSQL. Interface employé sécurisée (login/mot de passe). Fonctionnalités CRUD : ajout de pensionnaires, saisie de soins, transferts. Suivi des vaccins avec génération de listes de rappel.

Ce qui m'a appris le plus

Garantir l'intégrité des données au niveau de la base (contraintes, clés étrangères) et non seulement au niveau applicatif. Dissocier la logique métier des vues Flask.

vaccins à rappeler — requête SQL
SELECT a.nom, a.espece, s.date_soin, s.type_soin
FROM animal a
JOIN soin s ON a.id_animal = s.id_animal
WHERE s.type_soin = 'vaccin'
  AND s.date_soin < NOW() - INTERVAL '1 year'
ORDER BY s.date_soin ASC;

04 Algorithmique · C · Python · Analyse

Arbres Binaires
de Recherche

Projet allant au-delà de l'implémentation : vérification et génération d'ABR, analyse de complexité formelle, benchmarking sur des milliers d'arbres générés (presque complets, filiformes, quelconques), visualisations matplotlib et rapport complet avec preuves algorithmiques.

Ce que j'ai fait

Trois méthodes de vérification en C (naïve, définition optimisée, parcours infixe). Génération aléatoire d'arbres selon leur morphologie. Scripts Python de benchmarking et figures matplotlib automatisées.

Ce qui m'a rendu fier

Passer de « ça marche » à « je prouve pourquoi ça marche en O(n) et pas O(n²) », et confronter les complexités théoriques aux mesures expérimentales.

vérification ABR — O(n)
bool is_bst(Node *n, int min, int max) {
    if (!n) return true;
    return n->val > min && n->val < max
        && is_bst(n->left,  min,    n->val)
        && is_bst(n->right, n->val, max);
}
